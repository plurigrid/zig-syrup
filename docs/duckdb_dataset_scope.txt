NOTABLE DATASET CHARACTERISTICS
================================================================================
• claude-all-sessions: Deeply nested STRUCT types for message content,
  tool use, and agent data. Single denormalized table design.
  
• i.duckdb: Most diverse — 46 tables spanning agent ontologies, game theory,
  session analytics, SAW (self-avoiding walks), photonics, and transcripts.

• twitter.duckdb: SCD2 (slowly changing dimensions) pattern with version_id,
  valid_from, valid_to for temporal versioning.

• belief_revision.duckdb: ACSet-inspired schema modeling AGM belief revision
  postulates, epistemic entrenchment, and Grove sphere systems.

• basin-replay.duckdb: Session replay infrastructure with Blake3 CIDs,
  Merkle trees, Catalan-Doppler analysis, and vivarium/warehouse topology.

• social_graph.duckdb: 125K messages — largest message corpus by volume.

• playlist_transcripts.duckdb: 343 transcripts with embeddings (FLOAT[])
  and cluster assignments — ready for vector search.

• epstein.duckdb: Knowledge graph schema (persons, organizations, locations,
  events, claims, RDF triples) — mostly unpopulated scaffolding.
